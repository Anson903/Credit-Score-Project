Adaboost

param_grid <- expand.grid(mfinal = c(1, 5, 10),maxdepth = c(3, 5, 10))
control <- trainControl(method = "cv", number = 3)
train_model <- function(data, mfinal, maxdepth) {model <- boosting(y ~ ., data = data,boos = TRUE,mfinal = mfinal,control = rpart.control(maxdepth = maxdepth)); return(model)}
evaluate_model <- function(model, data) {pred <- predict.boosting(model, newdata = data)
accuracy <- sum(pred$class == data$y) / nrow(data); return(accuracy)}
best_model <- NULL
best_accuracy <- 0
results <- data.frame()
for (i in 1:nrow(param_grid)) {
  params <- param_grid[i, ];accuracy_list <- c()
  for (fold in 1:control$number) {
    set.seed(fold)
    trainIndex <- createDataPartition(train$y, p = .8, list = FALSE, times = 1)
    train_fold <- train[trainIndex,]
    test_fold <- train[-trainIndex,]
    model <- train_model(train_fold, params$mfinal, params$maxdepth)
    accuracy <- evaluate_model(model, test_fold)
    accuracy_list <- c(accuracy_list, accuracy)
  }
  avg_accuracy <- mean(accuracy_list)
  results <- rbind(results, cbind(params, accuracy = avg_accuracy))
  if (avg_accuracy > best_accuracy) {
    best_accuracy <- avg_accuracy
    best_model <- train_model(train, params$mfinal, params$maxdepth)
  }
}
results$mfinal <- as.factor(results$mfinal)
results$maxdepth <- as.factor(results$maxdepth)
ggplot(results, aes(x = mfinal, y = accuracy, color = maxdepth)) +
  geom_point(size = 3) +
  labs(title = "Grid Search Results for Boosting Model",
       x = "Number of Final Iterations (mfinal)",
       y = "Accuracy",
       color = "Max Depth") +
  theme_minimal()

print(paste("Best Accuracy: ", round(best_accuracy * 100, 2), "%", sep = ""))
## [1] "Best Accuracy: 68.25%"

A parameter grid was defined, containing the number of base classifiers (decision trees) mfinal, ranging from (1, 5, 10), and the maximum depth of the decision tree maxdepth, also ranging from (3, 5, 10). The model was then trained and evaluated for each parameter combination using 3-fold cross-validation. The accuracy of each cross-validation was recorded and averaged. We used the boosting() function to build the AdaBoost model.

Results - performance evaluation

By comparing four confusion matrix visualizations, we investigated the performance of each model based on their class predictions. For standard class, the random forest model performed the best, making the most correct predictions. For poor class, the random forest also performed the best. For the good class, logistic regression performed the best. Based solely on the confusion matrix, the random forest seems to be the best classifier for our data. However, the confusion matrix only provides a summary of predictions in matrix form and does not fully represent model performance. Therefore, we also consider F1 score, accuracy and precision. Random forest still performs the best in precision, while KNN achieves the best recall and F1 score. Overall, random forest and KNN appear to be the most appropriate models for this problem.

Discussion

During the model development process, we found that each model had its drawbacks in terms of hyperparameter tuning, environment setup, and runtime. The KNN model, while showing an easy environment setup process and relatively short runtime, contained only a single hyperparameter, which can imply limited model variation. Logistic regression, AdaBoost, and random forest involved correlated combinations of hyperparameters, leading to longer runtimes, which forced us to sacrifice some accuracy for faster execution. Additionally, when building the AdaBoost model, we encountered package downloading issues, with many functions used for AdaBoost being outdated according to our online research. Despite evaluating model performance through multiple measurements, F1 score is calculated from precision and recall, making direct comparisons between precision and recall somewhat meaningless. Our evaluation methods were unidirectional, focusing only on the confusion matrix and the F1 score, which could introduce bias when selecting the best model. In addition, data imbalance is one of the important reasons for affecting the performance of the model. Although we have undersampled the data during data preprocessing, the standard class still occupies the majority, because we use cross-validation to improve the performance of the model and use random forest to reduce the influence of the main classes. For the next project, we agreed that we should have a better time management and distribute the work more efficiently by assigning one model to each team member, instead of asking three members to finish all the coding part. Additionally, we should include more interactive evaluation methods for comparing models. According to a report underlining the advantage of the advantages of the Matthews correlation coefficient (MCC) over F1 score and accuracy in binary classification evaluation [1], it has been noted that both F1 score and accuracy can generate misleading results on imbalanced datasets, similar to our dataset. Using MCC can address this issue. Therefore, we should consider using MCC for evaluation next time.

Conclusion

To conclude, based on our evaluation measurements, KNN showed the best performance in terms of F1 score, while random forest performed the best in terms of accuracy. The models we trained can be used as a reference for financial institutions and other organizations to evaluate client credit. By using machine learning algorithms, institutions can make data-driven decisions, which can lead to more informed credit decisions and ultimately, better financial outcomes for both the institution and its clients. Considering the complexity of the models, random forest demonstrated the best predictive power with 77% accuracy and an F1 score of 0.65, which is only 0.02 lower than KNN but enjoys a 7% higher accuracy. Additionally, the performance of the KNN model is greatly impacted by imbalanced class distribution, whereas random forest can effectively handle imbalanced data by adjusting the weights assigned to each class. Therefore, we choose random forest as the best model for this classification problem.
